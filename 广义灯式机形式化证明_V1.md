# 广义灯式机的形式化证明

## 论全双工对抗、可审查切片与信息输入依赖停机

---

### 摘要

本文在狭义灯式机的基础上定义广义灯式机：一种由两台狭义灯式机作为全双工对抗算子构成的开放系统。狭义灯式机的目标态为闭合切片，广义灯式机的目标态为可审查切片——ERROR 作为合法输出的可寻址部分。本文证明信息输入依赖定理（不增加新责任的输入等效于无输入，必导致停机），证明纯算子的物理非法性，并论证灯式机作为独立机器类的不可替代性。

---

### 1. 引言

狭义灯式机（前文 V3）在固定作用域、固定阶段、固定分辨率下，证明了结构损恒零与有限步停机。其形式化将正算子 $P$ 与负算子 $N$ 处理为函数签名：给定输入，必然返回输出。

现实中的对抗算子具有内部结构。它有自己的前提、条件树与停机条件，可能内部耗竭，可能与对端前提矛盾。函数签名不足以描述两台各自可能独立停机的机器之间的交互。广义灯式机处理的正是这一层问题。

本文只涉及广义灯式机的定义、停机学与接口协议。狭义灯式机的全部定理作为构件直接引用，不重新证明。退相干、复相干、淘汰等完整动力学留待后续工作。

---

### 2. 狭义灯式机回顾

三元组 $L=(\Gamma,P,N)$。权威切片 $\Gamma_t=(\Pi_t,T_t)$。公理 I（不可逆产出）与公理 II（有限有损观测）。引理 1（因果单向性）、引理 2（前提先在性）。停机定理、结构损恒零定理。详见前文，此处从略。

狭义灯式机的合法实例条件：目标态 $g^*$ 在固定作用域与固定分辨率下有限步可达。

---

### 3. 广义灯式机的定义

#### 3.1 四元组

$$L_G = (\Gamma_G,\; L_\alpha,\; L_\beta,\; \mathrm{obs}).$$

$\Gamma_G$ 为广义切片。$L_\alpha,L_\beta$ 为两台狭义灯式机（全双工对抗算子）。$\mathrm{obs}$ 为外部观测接口。

#### 3.2 广义切片

$$\Gamma_G = (\Pi_G,\; T_G,\; E_G).$$

$\Pi_G$：广义前提集，$\Pi_G = \Pi_\alpha \cup \Pi_\beta$。$\Pi_\alpha$ 与 $\Pi_\beta$ 中的矛盾项写入 $E_G$。

$T_G$：广义条件树，节点定义同狭义灯式机。

$E_G$：ERROR 集。每条 ERROR 可寻址，含节点地址与理由。ERROR 为合法输出的组成部分。

#### 3.3 对抗算子

$L_\alpha$ 与 $L_\beta$ 各自为狭义灯式机，满足 V3 全部定义。算子角色不由内部倾向预先指定，由每轮比较结果决定。

物理上合法的算子必须结构完整（定理 2，§5）。灯式机至少包含两个算子。

#### 3.4 演算

单轮演算为原子操作：

1. 读取启动时的完整切片 $\Gamma_G$；
2. $L_\alpha$ 与 $L_\beta$ 各自独立对当前输入运行至内部停机；
3. 互相输出结构，比较等价性；
4. 保留等价部分，写入 $\Gamma_{t+1}^G$；
5. 不等价部分合并双方理由，作为 ERROR 写入 $E_G$，请求拉取新事实；
6. 若理由不可分（双方均无法给出不等价的原因），判定当前输入不合法。

运行期间不接受新输入。轮次间到达的新观测作为下一轮输入。若推演过程中输入变化，以合并后的完整输入重新推演。

#### 3.5 可审查条件

$\Gamma_G$ 可审查当且仅当：每个节点五元组可寻址，每个 ERROR 可定位且含理由，每个责任域划分状态可读。

#### 3.6 无状态性与幂等性

灯式机不储存运算状态。每次启动读取完整切片。输入相同且算子结构一致则产出一致。

灯式机的宿主实现可在内部缓存推演过程以减少计算消耗，缓存属于宿主层优化，不属于灯式机原语。对外仅暴露单次完整输出。

#### 3.7 灯式机的必要条件

输出须包含高维结构（$\mathrm{addr}$ + $\mathrm{domain}$ + 互斥完备划分）。不输出高维结构的系统不满足灯式机定义。原语保留高维结构，不保留演算过程。

#### 3.8 灯式机之间的关系

引用，非修改。上游冻结切片为下游输入，下游不修改上游。

---

### 4. 停机

#### 4.1 审查停机

$\Gamma_G$ 满足可审查条件。输出 $\Gamma_G$，$L_G$ 生命周期终止。$\Gamma_G$ 可含 ERROR。后续处理 = 启动新的 $L_G'$，以 $\Gamma_G$ 为输入。

#### 4.2 耗竭停机

外部比对连续输出同构，无新责任产出。暴露给外层，请求新观测。

#### 4.3 饥饿停机

从未产出非空等价部分。非法实例。

---

### 5. 定理

#### 定理 1（信息输入依赖）

不增加新责任的输入等效于无输入，必导致停机。

*证明。* 由公理 II，有效观测注入 $F_{t+1} \supsetneq F_t$。有效观测须使至少一个节点的 $\mathrm{domain}$ 被扩展或细分（新责任产生）。

若无新责任，两端算子的输入不变。输入不变则各自内部停机的产出不变。产出不变则互相比较的等价部分不变。等价部分不变则 $\Gamma_{t+1}^G$ 与 $\Gamma_t^G$ 同构。

由狭义停机定理（同责变换为可逆操作，公理 I），同构的连续产出不构成权威推进。有限步后 $D_t^G = \varnothing$，停机。$\square$

#### 定理 2（纯算子非法）

物理上合法的算子必须结构完整，即同时具备生成与裁剪能力。

*证明。* 分两种退化情况。

纯正算子：只执行可逆操作，永远不支付不可逆代价。由公理 I，可逆操作不构成权威变更，系统永远不产出权威切片。同时，永远不支付 $kT\ln 2$ 的擦除代价等价于热力学第二定律所禁止的永动机。非物理态。

纯负算子：只执行不可逆裁剪，但无候选可供裁剪。由公理 II，系统无法从外部获取新信息（无观测注入候选层）。无热涨落的纯擦除态等价于热力学第三定律所禁止的绝对零度。非物理态。

两种退化均导致非物理态。因此合法算子须同时具备生成与裁剪，即具有完整的狭义灯式机结构。$\square$

#### 定理 3（灯式机的独立机器类地位）

灯式机与图灵机图灵等价（计算同样的函数），但为独立机器类。

*证明。* 图灵机可在纸带上写出与灯式机输出字节相同的 if-tree 文本（模拟成立）。需证明模拟不等于实现。

区分在于结构保证的层级。

图灵机的原语为读、写、移动与有限状态迁移。纸带上编码的 $\mathrm{domain}$ 与 $\mathrm{addr}$ 是数据，不参与状态转移函数。互斥完备划分作为校验逻辑存在于程序层，可被绕过（程序缺陷、篡改、跳过校验）。结构损恒零是程序的后置条件，依赖程序员保证。

灯式机的原语包含 $\mathrm{addr}$ 与 $\mathrm{domain}$。互斥完备划分是机器定义的组成部分，不可绕过。结构损恒零是机器的不变量，由定义保证。

不变量与后置条件的区别：不变量由机器在每次合法操作中自动维持，后置条件由外部保证且可失效。

类比：图灵机可模拟冯诺依曼机的全部行为，但冯诺依曼机为独立机器类——其地址寻址与跳转在硬件层保证正确性，图灵机的纸带编码不提供此保证。灯式机相对于冯诺依曼机的关系同构：责任域与寻址结构在原语层保证，冯诺依曼机的注释层不提供此保证。

三类机器按原语保证的维度递增：$a$（顺序流）$\to$ $a{+}b$（顺序流 + 实现索引）$\to$ $a{+}b{+}c$（顺序流 + 实现索引 + 权威支撑证书）。图灵等价保证计算能力相同，维度递增保证结构保证层级不同。

因此灯式机为独立机器类。$\square$

### 6. 接口协议

#### 6.1 输出合约

算子内部停机时须向广义层输出结构。收敛停机输出结果。耗竭停机输出 ERROR，含节点地址、耗竭原因与内部切片状态。

#### 6.2 前提暴露合约

算子须暴露 $\Pi_i$。广义层检测 $\Pi_\alpha$ 与 $\Pi_\beta$ 之间的矛盾并写入 $E_G$。

#### 6.3 活性合约

至少一轮比较须产生非空等价部分。否则为饥饿停机，非法实例。

#### 6.4 同责循环检测

外部比对连续输出是否同构。同构即耗竭停机信号。检测由外部执行，不依赖算子内部自检。

#### 6.5 自我结构审查

外部观测允许灯式机输出自我结构与算子切片，分别接受审查。

### 7. 与狭义灯式机的关系

| | 狭义灯式机 | 广义灯式机 |
|---|---|---|
| 算子 | 函数签名 $P,N$ | 狭义灯式机 $L_\alpha, L_\beta$ |
| 目标态 | 闭合切片（$D_t=\varnothing$, $\Gamma_t \cong g^*$） | 可审查切片（ERROR 为合法输出） |
| ERROR | 阻塞，须解决 | 合法输出，须可寻址 |
| 作用域 | 固定 | 可由 $\mathrm{obs}$ 扩展 |
| 生命周期 | 到达 $g^*$ 或耗竭 | 输出即停机，重启为新实例 |
| 演算 | P 释放，N 裁剪 | 双端独立输出，比较等价性 |
| 状态 | $\Gamma_t$ | $\Gamma_G$，无运算状态 |

狭义灯式机为广义灯式机的构件。广义灯式机的每一轮等价于两台狭义灯式机各自完成一次完整生命周期后比较产出。

---

### 8. 结论

广义灯式机将狭义灯式机的算子从函数签名提升为结构完整的机器。其目标态从闭合切片扩展为可审查切片，ERROR 成为合法输出的可寻址部分。

信息输入依赖定理表明：灯式机的运行依赖外部观测持续注入新责任，等效不输入则必停机。纯算子非法定理表明：物理上合法的算子须同时具备生成与裁剪能力，即具有完整的灯式机结构。灯式机独立机器类定理表明：图灵机可模拟灯式机输出但无法实现灯式机的原语级结构保证，三类机器的区分在于原语保证的维度递增。

广义灯式机的每一轮为原子操作，输出即停机。灯式机之间以引用关系连接，上游冻结切片为下游输入。灯式机不储存运算状态，不保留演算过程，只保留高维结构。
